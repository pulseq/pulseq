New features and performance improvements in the current development branch

April 2026

Please check out the current development branch, it contains substantial performance improvements and some exciting new features. Amongst them, a substantial speed-up of the basic event library infrastructure and seq.testReport(), more consistent behavior of function that modify Pulseq objects (e.g. mr.align() or mr.scaleGrad()), allowing to detect programming errors and performance penalties. We now also introduce an initial version of the seq.autoLabel() function, which is able to add labels to a many of Cartesian sequences and thereby make them compatible with the integrated image reconstruction on the scanner.

We intend to publish these features in the main branch by the end of April and will greatly appreciate your feedback!

Join us in our monthly Pulseq user group meeting

April 2026

As many of you already know, we hold our monthly Pulseq user group meetings as a monthly webinar. Unless there is a collision with international bank holidays, major conferences or alike, we meet for a hour on the first Friday of the month at 3:00 pm UTC. Anyone interested is welcome to join us! March 2026: new Pulseq Siemens interpreter bug fix release

April 2026

We have made substantial progress in resolving the infamous table undocked problem (scanner controller crash and restart) and release a patch under a version number v1.5.2p1. This version is a pure bug-fix release and does not contain any new features. In addition to fixing (hopefully all manifestations of the table undocked problem, but at least their substantial part), there is a fix included to setting the SET counter limit, as well as a fix for importing stored older Pulseq protocols leading to an incorrect setting of the "ADC length per segment" parameter on the sequence-special card.
